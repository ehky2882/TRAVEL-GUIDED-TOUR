#### Cluster smoothness (closes item #6 of the original brief)

The original brief said *"when panning without zooming, clusters
change — owner wants them stable across pan."* In session 15 I
read the code and reported that it already uses an absolute grid
origin, so this *should* work. Owner saw the bug on device anyway.

Cause: cell pitch is derived from `region.span / cellsAcross`.
MapKit reports sub-percent drift on `region.span` when a pan
gesture settles even without zoom — and any change in pitch
re-buckets markers that sit near a cell boundary, which is the
visible "clusters changing on pure pan" symptom.

Fix: added `HomeMapSection.snappedSpan(_:)` static helper that
rounds the span to two significant figures before cell pitch is
computed. Sub-percent drift collapses to a single value; real
zoom changes (always at least several percent per pinch step)
still cross snap boundaries cleanly and re-cluster as expected.

E.g. `0.005001 / 0.00499 / 0.005` all snap to `0.005`; `0.006`
remains distinct. Pin diameters were already constants across
zoom (StopPin 14 / 18 pt, ClusterPin 26 / 30 / 34 pt by count);
no change there.
