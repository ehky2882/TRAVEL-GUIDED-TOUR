## Sync (#279 library+makers, #287 recently-viewed, #283 logout-clear)

**`Data/SyncService.swift`** (`@MainActor @Observable`). Model: on sign-in
**pull → merge (union) → push**; while signed in **write-through** each local
change, debounced 2s, as a full-state replace (upsert + `delete-not-in`) via
supabase-swift PostgREST. Anonymous = on-device only, unchanged.

- Stores (`LibraryStore`, `SavedMakersStore`, `RecentlyViewedStore`) each got
  an `onChange` write-through hook + `applyMerged(_:)` (sets merged state
  without re-firing the hook). Wired in the App entry's `.task` (captures the
  live instances).
- **Merge is pure + unit-tested** (`SyncServiceMergeTests`, 7 cases): union;
  saved/completed survive from either side; `listenedSeconds` max, `lastListened`
  later; **`downloadedAt` stays device-local**; recently-viewed keeps the
  latest `viewedAt`.
- **Logout clear (#283):** sign-out `applyMerged([])` on all three stores —
  **local-only, no remote delete** (restored on next sign-in). Anonymous users
  never fire it (nil→nil isn't an observable change).
- **`RecentlyViewedStore` reshaped (#287):** now stores
  `RecentlyViewedEntry{tourId, viewedAt}` (the table needs a timestamp).
  **Same UserDefaults key** — old `[String]` is read via `stringArray` and
  migrated; new format via `data`/JSON; the two are distinguished by value
  type, so migration is clean AND the existing store tests' key cleanup still
  isolates state. `tourIds` is now a computed accessor (Home rail unchanged).
- Tables (already applied via `accounts.sql`): `user_library`,
  `user_saved_makers`, `user_recently_viewed`; RLS own-row (`auth.uid()`).

## Recents-recording fix (#285, build 56)

**Bug:** the Library "Recents" list stopped updating for tours played via the
**mini-player**. **Root cause:** progress was recorded **only inside the
full-screen `PlayerView`** (its `.paused`/`.ended`/`.onDisappear`), but
"Start Tour" plays through the mini-player and does NOT auto-open the full
player — so listening without expanding never recorded anything. **Not
sync-related** (affects anonymous too; only *saving* was fine).

**Fix:** moved recording into **`AudioPlayerService`** (owns state +
`currentSourceId` = the tour + `currentTime`): new `onProgressCheckpoint`
hook fires on **pause / end / stop / interruption** regardless of UI, wired at
app start to `LibraryStore.updateProgress`. Removed the redundant
`writeProgress` from `PlayerView` (kept its end-of-playback handling). Bonus:
`.ended` passes `completed:true` → finished tours are now marked completed
(the old code always wrote `completed:false`).
