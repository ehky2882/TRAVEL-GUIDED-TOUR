## The big change — player presented from the top window

Earlier the bottom-module **window** was hidden while the player (a
`.fullScreenCover` in the main window) was up, because that window sits
at `windowLevel = .normal + 1` (above modals) and would otherwise float
over the player. That hide/show could never be timed to the cover's
slide, so the transition flashed a gap where the module should be
(worse on present — the cover reaches the bottom area **last**).

**Fix:** present `PlayerView` as a `.fullScreenCover` **from
`BottomModuleRoot`** (the top window) instead of `ContentView`. The
cover slides up over the mini-player + tab bar **in the same window** —
they're covered/revealed naturally by the cover's own animation, zero
hide/show, zero timing. Removed `BottomModuleWindowController.setHidden`
and the App-level `onChange(showingFullPlayer)` toggle.

`PassThroughWindow.hitTest` now returns `super.hitTest` (claims every
touch) whenever `rootViewController?.presentedViewController != nil` —
otherwise the pass-through rule (only the bottom strip is interactive)
leaves the full-screen player untappable above that strip.

## Island-vs-edge on retract

Tours start from their detail sheet (`tourPresenter.present`, which
stays open). Tapping the mini-player opened the player over it, so
retracting landed back on the still-open detail → bottom module forced
to edge-to-edge. **Fix:** opening the player dismisses the detail
(`ContentView` `onChange(showingFullPlayer)` → `tourPresenter.dismiss()`),
so retract returns to the tab root — Home shows its floating island.
`BottomModuleRoot` also reads `showingFullPlayer` so geometry recomputes
on toggle.
