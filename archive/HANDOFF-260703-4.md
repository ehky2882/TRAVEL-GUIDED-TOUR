## The five items

1. **🐛 Audio/transcript writes were broken** — `"failed to parse order (eq.0)" (line 1, column 4)`. The `stops` table has a column literally named **`order`**, which collides with PostgREST's reserved `order` sort param: `.eq("order", value: 0)` → `?order=eq.0` → HTTP 400. Proven via curl (`stops?order=eq.0` → 400; `stops?tour_id=eq.<uuid>` → 200). Fix: single-stop drafts, so `attachAudio` / `stopTranscript` / `setTranscript` filter by `tour_id` alone. **Shipped broken in builds 62/63** (login-only write path the sim can't reach). Memory: `reference-postgrest-order-column-collision`. **When multi-stop authoring lands, filter by the stop `id` — never re-add `.eq("order", …)`.**
2. **Editor Save unreachable** — `ProfileEditorView` ScrollView ran under the mini-player + tab bar; added the `.safeAreaInset(.bottom, AtlasBottomModule.height())` it was missing (also added to `CreateTourView`).
3. **Avatar pinch-zoom crop** — new `Features/Profile/AvatarCropSheet.swift`: circular viewport, pinch (1–6×) + drag, "Use photo" renders the visible square to 512² JPEG via `ImageRenderer` (same `imageLayer` for preview + render → WYSIWYG). Replaces the auto centre-crop; `ProfileEditorView` opens the sheet on pick.
4. **Discard-changes prompt** — closing the editor dirty asks "Discard changes?" (Discard / Keep editing); `hasChanges` diffs every field; `.interactiveDismissDisabled(hasChanges)` blocks swipe-bypass.
5. **New Tour → editor handoff** — button "Save draft" → **"Save draft & continue"** + a "Step 1 of 2. Next, you'll add audio, photos, and a transcript." caption; `CreateTourView` returns the new draft id via `onCreated`, and `MakerView` pushes `TourAuthoringView` once the create sheet fully dismisses (`pendingDraftId` + `.onChange(showingCreate)` avoids the dismiss↔push race; `navigationDestination(item:)` with a small `EditingDraft: Identifiable, Hashable` wrapper).
