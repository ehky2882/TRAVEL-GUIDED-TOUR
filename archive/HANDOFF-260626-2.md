## Root cause

`CLLocationManager` only delivers `didEnterRegion` on a boundary
**crossing** — never for a region the user is already **inside** when
monitoring begins. In `Tours.json`, AMNH's intro (stop 1, `manual`) and
stop 2 (`geofenced`, 30 m) share **identical coordinates**
(`40.78083090443824, -73.97279862427825`), so the user begins inside
stop 2's region at tour start → no entry event → stop 2 never plays.
Stops 3/4/5 are 150–250 m away, so they cross normally.

**This is general, not AMNH-specific** — any tour where the user starts
inside the first geofenced stop hits it. **4 of 5 multi-stop tours lead
with a geofenced stop-0** (Fifth Avenue Walk, After the Fire, Albertopolis,
The Spine of Power; AMNH is the exception with a manual intro). The fix is
in **code, not tour data** — the AMNH shared coords just make it a reliable
repro.

## What changed

**`Location/ProximityMonitor.swift`** (the bulk):
- After registering each `CLCircularRegion`, call `manager.requestState(for:)`
  and implement `locationManager(_:didDetermineState:for:)`. An `.inside`
  determination is treated as an arrival.
- **Defer vs. interrupt:** pure helper `decideInsideStopAction(insideStopIds,
  orderByStopId, playedStopIds, isPlayerBusy) -> .doNothing/.playNow/.waitForCurrentToEnd`.
  Player idle → play now; player busy (intro running) → hold the stop and
  fire it when the current item ends, watched via `withObservationTracking`
  (the monitor isn't a SwiftUI view). The intro is never cut off.
- **De-dupe** via a `playedStopIds` set — a stop played by a real
  `didEnterRegion` won't replay from `.inside`, and vice-versa.
- **Overlapping regions at start** → pick the **first stop in tour order**
  (lowest `order`); ties break by id. Documented in the helper.
- `stopMonitoring()` now resets the new state (`playedStopIds`,
  `insideStopIds`, `pendingInsideStopId`, `isObservingForPendingFire`).

**`Features/Player/PlayerView.swift`** (small touchpoint):
- Passes `startedStopId: appShared.currentPlayingStopId` into
  `startMonitoring(...)` so the monitor seeds the stop the UI started
  directly (a geofenced stop-0 with no intro) as played → never
  double-fired.
- Calls `proximityMonitor.cancelPendingInsideStop()` in the intro→stop-0
  auto-advance branch of `handlePlaybackEnded()`, so the UI and the monitor
  don't both start stop 0.

`AudioPlayerService` unchanged — the monitor reuses the existing
`handleEntry` → `audioPlayer.play(...)` path (no new AVPlayer), and reads
`audioPlayer.state` for busy/idle.

## How the three multi-stop shapes resolve

- **AMNH** (manual intro = stop 0; stop 2 geofenced at start coords):
  `startedStopId` = the manual intro (not monitored). Stop 2 pends while the
  intro plays, fires when it ends. ✅ (the bug)
- **Intro-audio tours** (Fifth Avenue Walk — real `introAudioURL`, stop 0
  geofenced): `startedStopId` = nil. Stop 0 pends; at intro end the UI
  auto-advances to stop 0 **and** calls `cancelPendingInsideStop()`, so only
  the UI starts it. No double audio.
- **No-intro geofenced-stop-0 tours** (After the Fire, Albertopolis, Spine):
  UI plays stop 0 immediately on open → `startedStopId` seeds it as played →
  monitor skips it.

## Verification

- **Unit:** new `ProximityMonitorInsideStopTests` (7 cases over the pure
  helper). `decideInsideStopAction` is `static`, takes only `UUID`/dict/`Set`/
  `Bool`, so no `CLLocationManager` or `Stop` construction needed.
- **`test_sim`: 110/110** (103 prior + 7). CI green (Build + unit tests +
  Validate Tours.json).
- **Sim repro (iPhone 17 Pro, iOS 26.5):** `set_sim_location` to
  `40.78083, -73.97280` **before** Start → opened Four Facades → Start Tour →
  opened the full PlayerView (monitoring starts in its `onAppear`) →
  **stop 2 "Central Park West — Theodore Roosevelt Memorial" auto-fires with
  no movement.** Then `set_sim_location` to stop 3
  (`40.780246, -73.974889`) → **player advances to "West 77th Street —
  Original Main Facade"** on the normal crossing. Screenshots in the session
  scratchpad (`repro-stop2-fires.jpg`, `repro-stop3-crossing.jpg`).

  *Note:* geofence monitoring only starts when the **full PlayerView**
  appears (`startGeofenceMonitoringIfNeeded` in its `onAppear`) — the
  detail-sheet inline "Start Tour" alone does not arm it. The repro path is
  Start Tour → tap the mini-player to open PlayerView.
