## What happened this session

### 1. Six home-screen polish PRs

All small, focused, owner-driven turn-by-turn at the simulator.

- **[PR #128](https://github.com/ehky2882/TRAVEL-GUIDED-TOUR/pull/128) — muted standard map style.** `MapStyle.standard` now uses `.standard(emphasis: .muted)`. Hybrid + Imagery unchanged (no muted variant exposed). Map reads as a desaturated canvas so pins / placecard / chrome don't compete with the map's own colour.
- **[PR #129](https://github.com/ehky2882/TRAVEL-GUIDED-TOUR/pull/129) — pin sizes + cluster count typography.** `StopPin` diameter 14 / 18 → **16 / 20 pt** (unselected / selected). Cluster count text `.system(size: 12, weight: .semibold)` → `.system(size: 12, weight: .regular, design: .monospaced)` — matches the SF Mono editorial voice now used across the home caption surfaces.
- **[PR #130](https://github.com/ehky2882/TRAVEL-GUIDED-TOUR/pull/130) — curated POI categories.** New `HomeMapSection.tourPOI` static `PointOfInterestCategories.including([…])` allowlist. Visible: landmarks / museums / national monuments / libraries / castles / fortresses / theaters / movie theaters / music venues / stadiums / aquariums / planetariums / zoos / amusement parks / parks / national parks / beaches / marinas / universities / airports / public transport / hotels / parking / EV chargers. Hidden: ATMs, gas stations, pharmacies, banks, retail, nightlife, restrooms, sport venues, the entire activity-venue group. Easy to revisit — single list.
- **[PR #131](https://github.com/ehky2882/TRAVEL-GUIDED-TOUR/pull/131) — drawer list scoped to map view.** `displayedTours` previously sorted all `filteredTours` by distance from `userLocation`. Now filters to tours whose stops fall inside `sharedState.visibleRegion` and sorts by tour-centroid distance from the map *center*. Header count collapses to `displayedTours.count` so "N TOURS IN VIEW" matches the number of cards below; the strip-clipping `toursInViewCount(in:)` helper + `currentScreenHeight()` shim + unused `UIKit` import are gone. Doc comment flags the future rails direction: when this single feed splits into rails, this stays as the "Tours in map view" rail and a sibling rail sorting by user distance becomes "Near you" — both derivable from the same `filteredTours` + `visibleRegion` + `userLocation` inputs without a model change.
- **[PR #132](https://github.com/ehky2882/TRAVEL-GUIDED-TOUR/pull/132) — keyboard overlay fix.** `BottomModuleRoot` and `BottomSheet` both flipped from `.ignoresSafeArea(.container, edges: .bottom)` to `.ignoresSafeArea(.all, edges: .bottom)` so that focusing a `TextField` (e.g. inside SearchView) no longer pushes the bottom module + drawer up by the keyboard's height. The keyboard now slides up *over* them. Worth remembering as the standard pattern for any future surface that needs to stay anchored at the screen bottom — `.container` alone is not enough.
- **[PR #133](https://github.com/ehky2882/TRAVEL-GUIDED-TOUR/pull/133) — recenter on pin tap.** `onPinTapped` now animates `cameraPosition` to centre the tapped pin's coordinate. Preserves the current visible span (read off `sharedState.visibleRegion?.span`, fall back to `recenterSpan`) so the action reads as a pan, not a zoom. Pin sits at screen geometric centre; placecard rises above it.
