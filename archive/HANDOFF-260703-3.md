## C2 — custom profile icon
- New **`Components/MakerAvatarView.swift`** — single source of truth for maker avatars. Resolution: uploaded photo → emoji → custom initials+colour → initials derived from display name on a colour hashed from the id. A `Color(atlasHex:)` init parses `#RRGGBB`. Static `initials(for:)` / `color(for:)` / `palette` (10 hex swatches) so other renderers reuse the logic.
- Reused in: `MakerView.avatar`, `MiniPlayerBar.authorIcon`, `LibraryView.makerRow`, `SearchView.makerRow`, and `MakerArtwork` (lock-screen monogram render).
- `Maker` gains optional `avatarInitials`/`avatarColor`; `MakerRow` encodes them (explicit null). `MakerProfileService.saveProfile` takes the avatar fields; new **`uploadAvatar(_ jpeg:)`** puts a square JPEG at `tour-images/{maker_id}/avatar-{uuid}.jpg` (leading maker-id satisfies the storage RLS; unique name dodges CDN cache) via `ensureMaker()`.
- `ProfileEditorView` avatar section: live preview (picked photo or `MakerAvatarView(previewMaker)`), **Upload a photo** (`PhotosPicker` → `squareJPEG` aspect-fill crop to 512²), **or** initials (≤2, auto-caps) + a 10-swatch colour grid. `usingPhoto` = a picked image or a kept `avatarURL`; on Save, upload first then `saveProfile(avatarURL: hasPhoto ? url : nil, initials/color: else)`.
- **All 9 seed studios have an emoji**, so their public pages are unchanged (verified against Tours.json).
