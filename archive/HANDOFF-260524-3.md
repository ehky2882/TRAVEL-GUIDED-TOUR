**Player-state hardening (added late in the session).** Three bugs
surfaced when the owner reviewed the running build:

1. `TourDetailView` — "Open player" was `.disabled(isThisTourLoading)`,
   greying out the primary action whenever AVPlayer was mid-load,
   including the stuck-loading state that followed a seek-to-end. The
   sheet is the user's escape hatch and must stay reachable.
   Disabled-gate removed.

2. `AudioPlayerService.seek(to:)` — now synthesizes `.ended` when the
   user scrubs to (or past) `duration`. `AVPlayerItem.didPlayToEndTime`
   only fires on natural end-of-playback, not on `seek(to: duration)`,
   so without this AVPlayer parks in
   `.waitingToPlayAtSpecifiedRate` → the timeControlStatus observer
   flips state to `.loading` → mini- and full-player stuck on
   hourglass / "Loading…". Synthesizing `.ended` mirrors natural-end
   semantics: pause player, snap currentTime to duration, cancel the
   loading watchdog, fire updateNowPlayingInfo. The existing PR #61
   `.ended` paths (mini-player replay, full-player play-icon) then
   "just work".

3. `PlayerView.togglePlayPause` — on `.ended` it was calling
   `audioPlayer.play()`, a no-op on the drained AVQueuePlayer queue.
   PR #61 added `replayLast()` for the mini-player but the full
   player was never wired. New `replayCurrent()` helper calls
   `playIntro` / `playStop` directly (bypasses the same-source guard
   in `startPlaybackIfNeeded` that would short-circuit a restart).

## Process notes / gotchas for next session

- **The polish branch was cut before PR #61 merged**, so PR #61's
  AudioPlayerService + MiniPlayerBar changes were absent from PR #60's
  initial diff. Branch needed rebasing onto current main before the
  three new audio-state fixes could be added cleanly. Lesson for the
  next branch-on-a-branch situation: rebase before reviewing.

- **XcodeBuildMCP's UI-automation workflow** (`tap`, `swipe`,
  `long_press`, `gesture`, etc.) was not enabled by default. Now
  configured in `~/.claude.json` via
  `XCODEBUILDMCP_ENABLED_WORKFLOWS=simulator,simulator-management,ui-automation,project-discovery,utilities`.
  Required a Claude Code restart to take effect. Without these tools
  the visual review can't drive drawer drags, navigate, or test
  playback paths.

- **AVPlayer note:** `seek(to: duration)` does **not** fire
  `AVPlayerItem.didPlayToEndTime` — that notification is reserved for
  natural play-to-end. Anything that needs end-of-tour semantics on a
  manual seek must synthesize the state transition itself (now done
  in `AudioPlayerService.seek(to:)`).
