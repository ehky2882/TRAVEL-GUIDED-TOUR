## What shipped

### PR #154 — Search polish
- **Typography → `caption`** across the search field, recent searches,
  and no-results copy. The **one exception**: result-row + maker-row
  **titles stay `body` ALL CAPS** (mirrors the Player's "stop titles →
  BODY all-caps"). The 36pt no-results magnifying-glass stays an
  illustration; the nav-bar "Search" title stays system font (chrome).
- **Result rows.** Title is single-line, all-caps, tail-truncated ("…");
  subtitle is **maker name only** (category + "•" bullet dropped);
  thumbnails get **square corners**. This removed the two-line
  category•maker wrap that read as a layout glitch.
- **Makers section.** When the query matches a maker name, a **MAKERS**
  group renders above **TOURS** (caption all-caps headers; tours-only
  queries stay headerless). Maker rows — circular emoji avatar, all-caps
  name, tour-count subtitle — **deep-link to `MakerView`** via a
  `NavigationLink` on the host nav stack (same pattern as "Go to
  creator").

### PR #160 — place-search performance
- **Root cause of the typing lag:** the place lookup ran a full
  `MKLocalSearch` (network round-trip) on **every keystroke**.
- **Fix:** `PlaceSearchService` rewritten around **`MKLocalSearchCompleter`**
  — Apple's type-ahead engine streams lightweight `PlaceSuggestion`
  (title/subtitle + the underlying completion) as the query fragment
  changes. The heavy `MKLocalSearch` now runs **once, on tap**
  (`resolve(_:)`), to get the coordinate/region the Home map flies to.
  Per-feature zoom logic (placemark `CLCircularRegion` radius) is kept on
  that resolve.
- **Tap feedback.** `SearchView` shows a `ProgressView` spinner on the
  tapped place row while it resolves, and ignores extra taps mid-resolve
  (also avoids back-to-back `MKLocalSearch` calls / rate-limiting).
- **Concurrency note:** `PlaceSearchService` is `@Observable` but
  **not** `@MainActor` — `MKLocalSearchCompleter` delivers its delegate
  callbacks on the main thread and every call site is main-isolated, so
  this keeps the delegate conformance from crossing an actor boundary
  (would be a Swift 6 error).

  map flies; the spinner covers it. In rapid automated testing the
  resolve occasionally returned nil (likely `MKLocalSearch` rate-limiting
  from hammering) — in normal one-tap use it's fine, and a nil resolve
  now just clears the spinner (no dead-end). Worth a calm device pass to
  confirm the fly lands on the chosen city.
